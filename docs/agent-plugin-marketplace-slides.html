A curated marketplace of agent plugins for AI coding clients (Cursor, VS Code Copilot, and similar).

Each plugin is a self-contained bundle that can include:

  • Agent definitions — e.g. agents/*.md
  • Skills — reusable instructions and scripts under skills/
  • MCP configuration — e.g. .mcp.json, optional server.json

Plugins live under plugins/<org>/<group>/<plugin-id> so many teams can contribute. This repository is the DCG marketplace — plugins shipped here today live under plugins/dcg/pse/ and are listed in .github/plugin/marketplace.json.

Where to host Git: GitHub Innersource vs Azure DevOps

Intel teams can host marketplace source on GitHub Intel Innersource or in Azure DevOps — different defaults for access and how easy it is for consumers to get started.

GitHub Intel Innersource

  • Intel default for Inner Source Git — familiar to most engineering teams.
  • Blue badge scenarios often do not require AGS group membership for standard access patterns (still follow your org’s identity policy).

Best when you want the broadest GitHub-native collaboration and Inner Source alignment.

Azure DevOps

  • Can reduce per-user Git steps — not everyone must clone the repo the way many GitHub-based setups assume during first-time setup.

Best when minimizing clone-and-config friction matters more than defaulting to GitHub.

Governance and access

Badge tiers are enforced outside this repo (hosting / identity). In-repo signals: owners + CODEOWNERS.

TopicMeaning
Blue badge Default baseline access for the marketplace or hosting surface.
Green badge Elevated access; granted via the AGS group, not automatic with blue.
Contributors AGS remains available when entitled (e.g. same group / approved paths), per org policy.
Plugin owner Every plugin has owners for approvals: marketplace.json owners + matching CODEOWNERS paths.

CODEOWNERS, reviews, and approvals

Pull-request routing in parallel with discovery: who GitHub asks to review, and how to keep registry and ownership aligned.

Git repository — pull requests

.github/CODEOWNERS

GitHub requests reviews from listed owners when matching paths change

  • Required reviewers for changes under each plugin directory.
  • Registry / meta — extra owners when marketplace.json or CODEOWNERS changes (see file comments).

Approval flow

  1. Contributor opens a PR under plugins/<org>/<group>/<plugin-id>/.
  2. GitHub applies CODEOWNERS — plugin owner(s) are requested.
  3. If the PR touches marketplace.json or CODEOWNERS, marketplace/repo owners are also required (see comments in CODEOWNERS).

New plugins

Keep owners in marketplace.json aligned with CODEOWNERS for that plugin path.
